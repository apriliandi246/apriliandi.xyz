My First Vscode Extension

We know that Vscode is the most popular text editor for programmers. A lot of features we can use in Vscode and one of my favorite features is Vscode extensions. We can use useful extensions for our productivity like code formatted, linter, syntax highlighting and more.

But, why not try to make our Vscode extension to solve our problem. Yeah, a couple of months ago I published my first Vscode extensions. It's just a simple extension but it solves my problem. Basically, my problem is when I write CSS code and after that, I read the code, but I'm more comfortable when the CSS properties are sorted, it makes me enjoy especially when I read the code.

Before :

				
.component {
   white-space: pre-line;
   display: block;
   resize: none;
   font-family: monospace;
   width: 100%;
   padding: 15px;
   border: 1px solid #38444d;
   font-size: 18px;
   outline: none;
   transition: background-color 0.2s;
   box-sizing: border-box;
   background-color: #253341;
   margin-top: 17px;
   color: #f5f5f5;
}

Smaller to Bigger :

				
.component {
   width: 100%;
   resize: none;
   padding: 15px;
   outline: none;
   display: block;
   color: #f5f5f5;
   font-size: 18px;
   margin-top: 17px;
   white-space: pre-line;
   font-family: monospace;
   box-sizing: border-box;
   border: 1px solid #38444d;
   background-color: #253341;
   transition: background-color 0.2s;
}

Before I create the extension, I have to manually sort all of the properties. And after that, I build a web application to handle this. But I'm so lazy to switch from the text editor to the browser and come back again to the text editor. That is why I build this extension.

You can check the code in my Github repository (the web version or the Vscode extension version). And you can see all my solutions for sorting CSS properties from this repository.
